# v0.7 Bug 修复：Sidebar/时间线/相册 7 项 UI 修复

## 背景

v0.7 集成时发现用户报告的 7 个 UI Bug 集中在 Sidebar、相册、时间线、右键菜单四大区域。逐一排查后整理如下。

## 修复一览

| # | Bug | 根因 | 修复 |
|---|-----|------|------|
| 1 | 收藏 section 计数不更新 | `onFavoriteChange` 已加 getFavorites 刷新，但 `isFav` 内部 state 不会跟随外部 `photo.is_favorite` 变化而重置 | 在 [PhotoCard.tsx](frontend/src/components/photos/PhotoCard.tsx) 加 `useEffect([photo.is_favorite])` 同步 |
| 2 | 时间线月/年视图无照片 | Backend 早返回 `cover_photo_path` 单一路径，前端只显示一张图（或没数据时只显示日期文字） | 后端 `get_timeline_*` 加 `cover_photo_paths` 数组（4 张），前端 TimelineView 实现 2x2 缩略图网格（带加载状态） |
| 3 | 时间线右键"加入相册"无反应 | TimelineView 没把 `onJoinAlbum/onDelete/onFavoriteChange/onRemoveFromAlbum` 透传给 PhotoGrid | 在 [TimelineView.tsx](frontend/src/components/timeline/TimelineView.tsx) 补齐所有 props |
| 4 | 加入相册弹窗粗糙 | 仅一个文字输入框 + 简单列表，缺封面、缺选中态、缺搜索、缺新建表单 | 重写 [JoinAlbumModal.tsx](frontend/src/components/dialogs/JoinAlbumModal.tsx)：3 列网格 + 封面缩略图 + 选中蓝色高亮 + 搜索框 + 新建表单 + 多选支持 |
| 5 | 相册默认封面缺失 | Sidebar 列表项用纯色占位、App.tsx 卡片视图 `cover_photo_id` 判断但没显示背景图 | 后端 `get_albums` 加 `cover_photo_path` 字段；Sidebar/App 都用 `cover_photo_path` 渲染缩略图，无封面时统一显示渐变占位（橙→红→紫）+ 山形图标 |
| 6 | 选文件夹时时间线 section 也被选中 | `currentView` 只有 timeline/favorites/albums/album-detail，选文件夹时复用 'timeline' | 扩展 `currentView` 增加 `'folder'`，在 [App.tsx](frontend/src/App.tsx) `handleSelectPath` 中设 `currentView: 'folder'`；Sidebar 同步更新类型 |
| 7 | 右键"在资源管理器中显示"无反应 | 后端 `/api/folders/open-in-explorer` 仅接受目录；文件路径会 400 | 改造 [open_folder_in_explorer](backend/api_server.py) 接受文件路径：Windows `explorer /select,<file>` 打开父目录并选中文件；macOS `open -R`；Linux 退化为打开父目录 |

## 修改文件

- [frontend/src/App.tsx](frontend/src/App.tsx) - `currentView` 加 'folder'，相册卡片封面渲染
- [frontend/src/components/layout/Sidebar.tsx](frontend/src/components/layout/Sidebar.tsx) - `currentView` 类型扩展，相册列表封面渲染
- [frontend/src/components/photos/PhotoCard.tsx](frontend/src/components/photos/PhotoCard.tsx) - isFav useEffect 同步
- [frontend/src/components/timeline/TimelineView.tsx](frontend/src/components/timeline/TimelineView.tsx) - 透传 props
- [frontend/src/components/dialogs/JoinAlbumModal.tsx](frontend/src/components/dialogs/JoinAlbumModal.tsx) - 重写 UI
- [backend/api_server.py](backend/api_server.py) - `get_albums` 加 cover_photo_path；`open_folder_in_explorer` 支持文件路径

## 验证

- ✅ `npm run build` 通过 (367KB JS)
- ✅ 后端 get_albums 返回 cover_photo_path
- ✅ 文件资源管理器打开：Windows `explorer /select,<file>`，macOS `open -R`
