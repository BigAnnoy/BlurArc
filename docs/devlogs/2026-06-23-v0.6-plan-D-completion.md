## 关键交付物

### 1. 原子计数器模块 `backend/dest_filename.py`

```python
import itertools
import threading

_seq_counter = itertools.count(1)  # 进程级原子计数器
_filename_lock = threading.Lock()   # taken_names 锁

def build_dest_filename(media_date, ext, month_dir, is_dup, taken_names):
    """O(1) 生成不冲突的目标文件名（线程安全）"""
    stem = media_date.strftime("%Y%m%d_%H%M%S")
    with _filename_lock:
        while True:
            n = next(_seq_counter)
            candidate = f"{stem}_{n:03d}{'_dup' if is_dup else ''}{ext}"
            if candidate not in taken_names:
                taken_names.add(candidate)
                return month_dir / candidate
```

**核心收益：**
- 消除 O(n) 文件系统探测（旧实现在 while 循环里反复调用 `dest.exists()`）
- 消除多线程 TOCTOU 重试（import_manager 的 file_lock 仍是首要保护）
- 10K 文件预估节省 5-20 秒

### 2. 集成到 `import_manager._import_file`

```python
# 之前：O(n) 文件系统探测
def _build_dest_filename(self, media_date, ext, month_dir, is_dup=False):
    base = media_date.strftime('%Y%m%d_%H%M%S')
    counter = 1
    while True:
        name = f"{base}_{counter:03d}{...}{ext}"
        dest = month_dir / name
        if not dest.exists():
            return dest
        counter += 1

# 现在：委托给原子计数器模块
def _build_dest_filename(self, media_date, ext, month_dir, is_dup=False, taken_names=None):
    if taken_names is None:
        taken_names = set()
    return _build_dest_filename_atomic(media_date, ext, month_dir, is_dup, taken_names)
```

调用处 `_import_file` 维护一个类级别 `_import_taken_names` 集合，在 file_lock 临界区内复用。
