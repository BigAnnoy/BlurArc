## P1-1：导入检查单次遍历 ✅

### 改动

[api_server.py L1503-1566](file:///e:/BlurArc/BlurArc/backend/api_server.py#L1503-L1566) 的 `_perform_import_check`：

**之前**（遍历两次目录）：
```python
# 第一次遍历：仅统计文件总数
all_files_total = sum(len(files) for _, _, files in os.walk(source_path))

# 第二次遍历：处理文件
for root, _, files in os.walk(source_path):
    ...
```

**之后**（单次遍历 + 动态进度估算）：
```python
scanned_files = 0
_ESTIMATED_MEDIA = 200  # 渐进公式参数

for root, _, files in os.walk(source_path):
    for filename in files:
        scanned_files += 1
        # ...处理文件...
        if scanned_files % 50 == 0:
            # 渐进公式：49 * n / (n + K)
            stage_progress = int(49 * scanned_files / (scanned_files + _ESTIMATED_MEDIA))
            emit(stage_progress, 'scanning', f'扫描中... 已扫描 {scanned_files} 个文件')
```

### 性能数据

| 指标 | 修复前 | 修复后 | 提升 |
|------|--------|--------|------|
| 目录遍历次数 | 2 | 1 | **50%** |
| `os.stat()` 调用 | N×2（扫描 + 处理） | N | **50%** |

from functools import lru_cache

@lru_cache(maxsize=500)
def _cached_video_metadata(file_path_str: str, file_mtime: float) -> dict:
    """缓存视频元数据，key 为 (路径, mtime)
    
    mtime 变化时缓存失效（用户可能替换了文件）
    """
    try:
        from .video_processor import VideoProcessor
    except ImportError:
        from video_processor import VideoProcessor
    return VideoProcessor.extract_metadata(file_path_str)


@app.route('/api/video/metadata', methods=['GET'])

### 当前 App.tsx 状态结构（待迁移）

[App.tsx L47-66](file:///e:/BlurArc/BlurArc/frontend/src/App.tsx#L47-L66) 的 18 个状态变量被分散在 4 个独立 `useState` 调用中。

需要分组到 Zustand store 切片：

| 切片 | 状态字段 |
|------|----------|
| `globalSlice` | `initialized`, `isFirstRun`, `loading`, `currentView` |
| `dataSlice` | `stats`, `years`, `rootDir`, `favoriteCount` |
| `photosSlice` | `photos`, `totalPhotos`, `currentPage`, `hasMore`, `selectedPath`, `selectedTitle`, `selectedAlbumId` |
| `selectionSlice` | `selectionMode`, `selectedIds` |

### 下一步行动

1. **创建 `frontend/src/stores/appStore.ts`**：使用 Zustand `create`，按上述 4 个 slice 拆分状态和 actions
2. **重构 `App.tsx`**：用 `useAppStore((s) => s.xxx)` 选择器替换 `state.xxx` 访问
3. **更新子组件 props**：Header / Sidebar / MainContent / TimelineView / PhotoPreview 等用 selector 订阅需要的字段
4. **验收**：构建通过 + 翻页时 Sidebar/Header 不重渲染（用 React DevTools Profiler 验证）

### 关键技术决策

- **使用 `useShallow` 或 selector 函数**避免浅比较失效
- **依赖注入**：`showToast` 和 `t` 通过 `setDeps()` 注入一次，避免每个 action 显式传参
- **API 调用保留在 store 内**：action 直接调 `api.*`，组件只触发 action

### 风险

- ⚠️ 工作量大（涉及 ~18 个 state 字段、~20 个 callback）
- ⚠️ 必须保持所有 useEffect 逻辑不变
- ⚠️ 子组件 props 接口不能变

### 推荐做法

- 使用 `using-git-worktrees` 创建独立工作区进行重构
- 完成后用 `test-driven-development` 写最小回归测试
- 严格按 P1 计划文档 §修复 6 的方案实施
