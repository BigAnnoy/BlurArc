## 0. 数据模型

v0.7 基于 v0.6 已有的 `Photo` / `Album` / `AlbumPhoto` 三张表，仅做字段补充，**不引入新表**。

### 0.1 现有表（已存在于 v0.6）

```python
# backend/database.py（节选）
class Photo(Base):
    __tablename__ = "photos"
    id = Column(Integer, primary_key=True, index=True)
    filename = Column(String(255), nullable=False)
    path = Column(Text, nullable=False, unique=True)
    size = Column(Integer, nullable=False)
    md5_hash = Column(String(32), index=True)
    media_date = Column(DateTime(timezone=True), index=True)
    file_type = Column(String(10), nullable=False)   # 'photo' | 'video'
    is_favorite = Column(Boolean, default=False)    # v0.6 已有

class Album(Base):
    __tablename__ = "albums"
    id = Column(Integer, primary_key=True, index=True)
    name = Column(String(100), nullable=False, unique=True)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
    description = Column(Text)

class AlbumPhoto(Base):
    __tablename__ = "album_photos"
    album_id = Column(Integer, ForeignKey("albums.id"), primary_key=True)
    photo_id = Column(Integer, ForeignKey("photos.id"), primary_key=True)
```

### 0.2 v0.7 新增字段

| 表 | 字段 | 类型 | 默认 | 索引 | 用途 |
|----|------|------|------|------|------|
| `photos` | `favorited_at` | `DateTime(timezone=True)` | NULL | ✅ | 收藏时间，用于"最近 7/30 天/一年"过滤 |
| `albums` | `cover_photo_id` | `Integer, ForeignKey(photos.id)` | NULL | — | 相册封面（v0.7 用第一张照片，**字段先建但 UI 暂不实现选择**） |
| `album_photos` | `added_at` | `DateTime(timezone=True)` | `now()` | — | 加入相册时间，用于"按加入时间"排序 |

### 0.3 索引与迁移

```sql
-- Alembic 迁移脚本（V0.7.0__add_favorite_and_album_meta.py）
ALTER TABLE photos ADD COLUMN favorited_at DATETIME;
CREATE INDEX ix_photos_favorited_at ON photos(favorited_at);

ALTER TABLE albums ADD COLUMN cover_photo_id INTEGER REFERENCES photos(id);

ALTER TABLE album_photos ADD COLUMN added_at DATETIME DEFAULT CURRENT_TIMESTAMP;
```

### 0.4 不引入的字段

| 字段 | 原因 |
|------|------|
| `is_pinned`（相册） | v0.7 不做相册置顶，§12 |
| `cover_photo_id` UI 写回 | v0.7 相册封面用第一张自动计算，UI 不暴露选择 |
| `is_screenshot` / `is_edited` | v0.7 不做，§12 |
| `pinned` | 同上 |

### 0.5 关联关系

```
Photo ──< AlbumPhoto >── Album
                            │
                            └─ cover_photo_id ──> Photo（自引用，可空）
```

一张照片可属于多个相册（多对多）；一个相册的封面来自关联照片中的一张（v0.7 取第一张）。

## 1. Sector 总览

v0.7 在 v1 的 sidebar 基础上新增了 5 个 sector，**统一 3 层布局**：

```
<sector>
  ┌──────────────────────────────────────┐
  │ <toolbar>     标题 + 通用操作 + 视图操作 │  ← 第 1 层
  │ <view-tabs>   视图专属 tabs            │  ← 第 2 层
  │ <content>     photo-grid / albums-grid │  ← 第 3 层
  │               / timeline-view         │
  └──────────────────────────────────────┘
```

| 顺序 | Sector | 触发 | 数据源 | 主内容 |
|------|--------|------|--------|--------|
| 1 | **⭐ 收藏** | 点击 sidebar | `photo.is_favorite=1` | 照片网格 |
| 2 | **🕐 时间线** | 点击 sidebar | 全部照片 | 按月分组瀑布流 |
| 3 | **📅 按年份浏览** | sidebar disclosure | `path=YYYY/YYYY-MM` | 照片网格 |
| 4 | **🖼 相册集** | 点击 sidebar | 全部相册 | 相册大卡片 |
| 5 | **📁 文件夹** | 点击 sidebar 任意子项 | 根目录 | 照片网格 |

### 2.3 Filter By 菜单（Apple Photos 风格）

打开后弹出 360px 宽的菜单，支持多选：

```
┌──────────────────────────────┐
│ 筛选                        │
├──────────────────────────────┤
│ ❤  全部收藏                ✓ │
│ 🖼  仅照片                    │
│ ▶  仅视频                    │
│ ───────────                  │
│ ⏱  最近 7 天                 │
│ ⏱  最近 30 天                │
│ ⏱  最近一年                  │
└──────────────────────────────┘
```

| Filter 项 | 含义 | 实现 |
|-----------|------|------|
| 全部收藏 | 取消所有筛选 | — |
| 仅照片 | `is_favorite=1 AND file_type='photo'` | 后端 query param |
| 仅视频 | `is_favorite=1 AND file_type='video'` | 后端 query param |
| 最近 7 天 | `favorited_at >= now - 7d` | 后端 query param |
| 最近 30 天 | `favorited_at >= now - 30d` | 后端 query param |
| 最近一年 | `favorited_at >= now - 1y` | 后端 query param |

## 9. API 端点总览

### 9.1 收藏

| Method | Path | 说明 |
|--------|------|------|
| `POST` | `/api/photos/{id}/favorite` | 收藏 |
| `DELETE` | `/api/photos/{id}/favorite` | 取消收藏 |
| `POST` | `/api/photos/batch-favorite` | 批量收藏 `{photo_ids: [..]}` |
| `GET` | `/api/photos?favorite=true&page=N` | 列出收藏 |

### 9.2 相册

| Method | Path | 说明 |
|--------|------|------|
| `GET` | `/api/albums` | 列出所有相册（带 count） |
| `POST` | `/api/albums` | 创建相册 `{name, description}` |
| `PATCH` | `/api/albums/{id}` | 重命名 / 改描述 |
| `DELETE` | `/api/albums/{id}` | 删除相册（CASCADE album_photo） |
| `GET` | `/api/albums/{id}/photos?page=N` | 列出相册内照片 |
| `POST` | `/api/albums/{id}/photos` | 批量加入 `{photo_ids: [..]}` |
| `DELETE` | `/api/albums/{id}/photos/{photo_id}` | 单张移除 |
| `POST` | `/api/albums/{id}/photos/batch-remove` | 批量移除 `{photo_ids: [..]}` |

### 9.3 照片（已有 + v0.7 新增）

| Method | Path | 说明 |
|--------|------|------|
| `GET` | `/api/photos?path=2024-10&page=N` | 按目录列出 |
| `GET` | `/api/photos?favorite=true` | 列出收藏 |
| `GET` | `/api/photos?sort=date_desc&group_by=month` | 时间线分组 |
| `PATCH` | `/api/photos/{id}` | 修改 metadata（path, is_favorite 等） |
| `DELETE` | `/api/photos/{id}` | 删除照片 |
| `DELETE` | `/api/photos?ids=1,2,3` | 批量删除 |

### 9.4 文件夹（v1 + v0.7）

| Method | Path | 说明 |
|--------|------|------|
| `POST` | `/api/import/scan` | 增量扫描 `{path: xxx}` |
| `GET` | `/api/folders` | 列出物理文件夹（v0.7 暂未实现） |

### 9.5 分页参数（所有列表接口通用）

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `page` | int | `1` | 页码，从 1 开始 |
| `per_page` | int | `100` | 每页数量，最大 `500` |
| `cursor` | string | — | 可选游标分页（用于时间线无限滚动，按 `media_date` DESC + `id` DESC 二元组） |

**响应格式**：
```json
{
  "items": [...],
  "total": 1234,
  "page": 1,
  "per_page": 100,
  "has_next": true
}
```

**时间线无限滚动**（§3.5）：
- 默认走 cursor 分页（`?cursor=2025-12-01T00:00:00_12345`）
- 每次返回 12 个月分组
- `has_next=false` 时停止加载

### 9.6 错误处理

所有 v0.7 新增 API 统一返回格式：

```json
{
  "error": {
    "code": "ALBUM_NAME_DUPLICATE",
    "message": "相册名称已存在",
    "details": {"existing_album_id": 5}
  }
}
```

| HTTP 状态码 | 场景 | code 示例 | UI 反馈 |
|------------|------|-----------|---------|
| `400` | 请求参数缺失 / 非法 | `INVALID_PARAM` | toast "请求参数错误" |
| `404` | 资源不存在（photo_id / album_id 无效） | `PHOTO_NOT_FOUND` | toast "照片不存在" + 刷新列表 |
| `409` | 相册名重复 | `ALBUM_NAME_DUPLICATE` | modal 内联错误，提示"该名称已存在" |
| `500` | DB 错误 / 文件系统错误 | `INTERNAL_ERROR` | toast "操作失败，请重试" + 错误日志 |
| `503` | 后台任务（扫描）进行中 | `IMPORT_IN_PROGRESS` | toast "导入进行中，请稍后再试" |

**前端统一拦截器**（`frontend/src/services/api.ts`）：
- 401 → 跳登录（v0.7 单机应用，理论上不出现）
- 404 → 从当前列表移除该 photo_id + toast
- 409 → 弹回 modal，不关弹窗
- 5xx → toast + Sentry 风格埋点（v0.7 简化为 `console.error` + toast）

## 13. 跨章约束

### 13.1 平台支持

**v0.7 仅支持 Windows 桌面端**（PyWebView + Flask）。

| 导出类型 | 范围 | 格式 | 实现 |
|---------|------|------|------|
| 导出原图 | 单张 / 批量 | 原始文件复制到目标目录 | 客户端 `shutil.copy2` + 选择目录 |
| 导出缩略图包 | 批量 | 缩略图打包为 ZIP | 复用 `thumbnail_manager` 现有缩略图 |

**不支持**（v0.7 范围外）：
- ❌ 视频导出
- ❌ 调色 / 滤镜 / 旋转后再导出
- ❌ HEIC → JPG 格式转换
- ❌ 邮件 / 网盘 / 第三方上传

### 13.3 性能预算

| 场景 | 目标 |
|------|------|
| 收藏视图首屏（10K 收藏） | < 500ms |
| 单个相册内照片列表（1K 照片） | < 300ms |
| "不在相册中" 过滤（20K 照片） | < 1s（**必须**用 LEFT JOIN，见 §4.4） |
| 时间线无限滚动每次加载 | < 800ms（12 月） |
| 加入收藏 / 取消收藏 API 响应 | < 100ms |
| 加入相册 modal 打开 | < 200ms |
