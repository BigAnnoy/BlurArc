|----------|----------|

| Pillow | 不 mock，用真实小图（< 1KB） |
| Dio (Flutter) | `MockDioHttpClientAdapter` |

## 14. 追加：导入性能深度优化（v0.6.0 同包发布）

> **目标**：在不动外部行为的前提下，把 1 万文件导入的耗时再砍 30-50%
> **范围**：仅 `backend/import_manager.py` 及其配套测试
> **理由**：测试深测会发现这些"看起来无害但累加可观"的冗余调用，提前修掉可大幅降低测试断言的等待时间

### 14.1 已识别的 8 个冗余点

| # | 位置 | 现状 | 影响（10K 文件） |
|---|------|------|------------------|
| 1 | `_scan_source` + `process_file` + except 块 | 每文件 `stat().st_size` 2-3 次 | +2-3 秒 |
| 2 | `_scan_source` 内层循环 | `for ext in MEDIA_FORMATS: endswith` 20+ 次/文件 | +2 秒（200K 次字符串比较） |
| 3 | `_import_file` 内 `SessionLocal()` | 每文件 1 个连接，建立/销毁 | +100 秒 |
| 4 | `_import_file` 内 `db.query(Photo).filter(...)` | 每文件 1 次 SELECT | +50-100 秒（10K 次查询） |
| 5 | `_get_media_date` 调用 PIL | 无 EXIF 文件也开 PIL | +300-500 秒（截图/网络图） |
| 6 | `_load_target_records` 全量加载到内存 | 10W 文件 = 10W 字符串 dict | +50MB 内存，扫描慢（**保留**，暂无优化） |
| 7 | `original_total_size` 循环 | scan 完后又一轮 `stat()` | +1 秒 |
| 8 | `_build_dest_filename` 序号探测 | 多线程并发探测同名 | +5-20 秒（TOCTOU 重试） |

### 14.2 优化方案

#### 优化 1：scan 时一并采集 size

```python
# 改动前：scan 阶段不采 size，process 阶段 stat
for file in files:
    if file.lower().endswith(media_ext):
        media_files.append(root_path / file)

# 改动后：scan 时用 os.scandir + stat_result 缓存 size
for entry in os.scandir(root):
    if not entry.is_file():
        continue
    ext = os.path.splitext(entry.name)[1].lower()
    if ext in MEDIA_EXT_SET:  # 见优化 2

```

#### 优化 4：EXIF magic bytes 快检

```python
# JPEG magic: FF D8 FF
# PNG magic: 89 50 4E 47
# HEIC magic: 00 00 00 ?? 66 74 79 70 68 65 69 63  ← 不一致
# 简单方案：文件头 4 字节白名单

EXIF_SUPPORTED_MAGIC = {
    b'\xff\xd8\xff',       # JPEG
    b'\x89PNG',            # PNG
    b'RIFF',               # WebP（前 4 字节）
    b'II*\x00',            # TIFF (little-endian)
    b'MM\x00*',            # TIFF (big-endian)
}

def _has_exif_magic(path: Path) -> bool:
    try:
        with open(path, 'rb') as f:
            return f.read(4) in EXIF_SUPPORTED_MAGIC
    except OSError:
        return False

# _get_media_date 前置
if not _has_exif_magic(file_path):
    return None  # 直接走 mtime fallback
```

**影响**：截图/网络图/无 EXIF 文件 -300-500 秒

#### 优化 6：原子计数器替代序号探测

```python
# 改动前：_build_dest_filename 探测文件系统
#   for n in range(1, 1000):
#       candidate = f"{stem}_{n:03d}{ext}"

# 改动后：进程级原子计数器
import itertools
_seq_counter = itertools.count(1)

def _build_dest_filename_fast(media_date, ext, month_dir, is_dup, taken_names):
    """O(1) 序号生成，要求调用方在锁内"""
    stem = media_date.strftime('%Y%m%d_%H%M%S')
    while True:
        n = next(_seq_counter)
        if is_dup:
            candidate = f"{stem}_{n:03d}_dup{ext}"
        else:
            candidate = f"{stem}_{n:03d}{ext}"
        if candidate not in taken_names:
            taken_names.add(candidate)
            return month_dir / candidate
```

**影响**：消除多线程 TOCTOU 重试，10K 文件 -5-20 秒
