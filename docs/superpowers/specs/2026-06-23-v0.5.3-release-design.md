## 三、CHANGELOG 编辑方案

在现有 `[0.5.3] — 2026-06-21` 章节下追加内容（**仅在已有子分类下加，不引入新分类**），保持 Keep a Changelog 格式 + 与现有 v0.5.3 一致。

> **注意：** v0.5.3 现有子分类为「新增 / 修复 / 变更」，本次不新增「性能」「清理」等新分类。性能改进归入"变更"，清理工作归入"变更"末尾。

### 3.1 修复 (Fixes) — 追加到现有 `### 修复` 末尾

- **移动端 UI 与原型对齐**（10 项修复）：暗色 AppBar/BottomNav 背景色、AppBar logo 尺寸、工具栏下拉箭头、上传按钮布局、Settings 主题选项顺序、Section header 平台差异化、Settings 区块标题（仅 mobile）、Folder 返回按钮、BlurArcLogo 默认色对比度优化
- **平板 UI 与原型对齐**（10 项修复）
- **平板模拟器 UI 修复**：导航栏/工具栏对齐
- **mDNS 广播修复**：`ServiceInfo` 参数顺序（`port` 必须在 `addresses` 之前）、自动启动、`wait_ready()` 确认
- **导入预检性能修复**：删除 `rglob` 兜底，目标目录去重改用集合，导入时只对大小相同的组计算 MD5
- **手机首屏秒加载**：`Dio` 添加 `sendTimeout` 防卡死、SQL 添加复合索引
- **移动端 6 Bug 修复**：Logo 资源使用错误、文件夹返回按钮文本（"返回相册"→"返回上级"）
- **代码审查 4 项修复**：
  - `home_page.dart` 断连页溢出保护（`SafeArea + SingleChildScrollView`）
  - `api_server.py` 缓存命中时复用 size，去掉 `stat()` I/O
  - `api_client.dart` `onDisconnected` 触发条件收敛（新增 `_everConnected` 标志）
  - `dev-start.bat` 抽取 `:deploy_avd_common` 公共函数

### 3.2 变更 (Changes) — 追加到现有 `### 变更` 末尾

**功能改进：**
- **设备信息服务**：新增 `DeviceInfoService` 缓存设备名称，避免重复 IPC
- **Android 高刷屏支持**：`MainActivity.kt` 添加 `Window.setFrameRate` API，90/120Hz
- **iOS 应用名称**：`AppDelegate.swift` 配置显示名 "Blur Arc"
- **导入去重 MD5 复用 DB 缓存**：`prescan_index` 改为 `(file, md5_hash, size)` 三元组，命中 DB 缓存时跳过 `stat()` I/O
- **手机端首屏秒加载**：照片列表分页 + 缩略图懒加载 + SQL 索引
- **导入预检并行化**：保持现有 `ThreadPoolExecutor` 并行 MD5 计算，新增"按文件大小分组后跳过不可能重复"的预筛

**UI 调整：**
- **Logo 资源统一管理**：从 SVG（`flutter_svg` 兼容性问题）改为 PNG 内置资源，新增 `title_bar_light.png` / `title_bar_dark.png`

- **暗色 AppBar 背景调整**：`darkBgCard` → `darkBgPage`（`#0c1117`），AppBar/BottomNav/平板工具栏统一透明
- **Section header 平台差异化**：mobile `14×12×8 + 13px`、tablet `16×16×8 + 14px`
- **Settings 主题选项顺序**：跟随系统 / 深色 / 浅色（原：跟随系统 / 浅色 / 深色）
- **手机上传按钮布局**：Column 上下堆叠（"开始上传"在上，"全部取消"在下）

## 四、Git 操作流程

### 4.1 提交 CHANGELOG 改动

```bash
# 1. 编辑 CHANGELOG.md（在 [0.5.3] 章节的「修复」「变更」末尾追加内容）
# 2. 提交
git add CHANGELOG.md
git commit -m "docs: 补充 v0.5.3 变更说明（移动端 UI + 性能 + 修复）"
```

**本次新增 commit 数：** 1
**推送前本地领先 origin/main 的总 commit 数：** 5（4 个已有 + 1 个 CHANGELOG）

### 4.2 推送到 origin/main

```bash
git push origin main
```

**风险评估：** 低。本地领先 5 个 commit，全部在 main 分支，远程 `origin/HEAD` 仍在 `15cbf03`。
**回退方法：** `git reset --hard origin/main`（不可逆，丢弃本地所有未推送 commit）

### 4.3 创建附注型 tag

```bash
git tag -a v0.5.3 -m "Release v0.5.3 — 移动端 UI 优化 + 性能修复 + Bug 修复

主要变更：
- 移动端 UI 与原型对齐（10 项修复）
- 平板 UI 与原型对齐（10 项修复）
- mDNS 广播修复（自动发现可用）
- 导入预检性能优化（去重 + 缓存复用）
- 手机首屏秒加载（SQL 索引 + 分页）
- 代码审查 4 项修复
- Logo 资源统一 + 设备信息服务
- dev-start 热更新选项（[9]/[10]/[5]）
- Android 高刷屏支持（90/120Hz）
- 删除临时脚本 check_logo.py

完整变更见 CHANGELOG.md"
```

**tag 消息格式：** 多行纯文本，与现有 `v0.5.2` tag 风格保持一致。

### 4.4 推送 tag

```bash
git push origin v0.5.3
```

**风险评估：** 低。tag 一旦推送即为公开版本（v0.5.2 已在 GitHub Releases 可见）。
**回退方法：** `git tag -d v0.5.3 && git push origin :refs/tags/v0.5.3`（删除 tag），不影响 commit。

### 4.5 （可选）GitHub Release

```bash
# 检查 gh CLI 是否已认证
gh auth status

# 已认证则执行（命令会从 CHANGELOG.md 截取 [0.5.3] 整段）
gh release create v0.5.3 \
  --title "Blur Arc v0.5.3" \
  --notes-file <(sed -n '/^## \[0.5.3\]/,/^## \[0.5.2\]/p' CHANGELOG.md | sed '$d')
```

**未认证则跳过本步。** 后续可在 GitHub 网页手动创建 Release。

## 六、风险与回退

### 风险表

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 推送冲突（其他人推送过） | 极低 | 中 | 单人开发，origin/HEAD 落后本地 4 commit；冲突时 `git pull --rebase` |
| Tag 命名错误 | 低 | 低 | 推送前 `git tag --list` 确认无重复 |
| CHANGELOG 内容漏改/写错 | 中 | 低 | 写入后 `git diff CHANGELOG.md` 自检；用 Keep a Changelog 格式 |
| `gh` 未认证导致步骤 4.5 失败 | 中 | 极低 | 跳过该步；不影响 tag 推送；后续手动补 Release |

### 回退方案

- **回退 commit 推送：** `git push --force-with-lease origin main`（仅在远端未被人新推时）
- **回退 tag：** `git tag -d v0.5.3 && git push origin :refs/tags/v0.5.3`
- **回退 CHANGELOG 改动：** `git revert <changelog-commit-sha>`
