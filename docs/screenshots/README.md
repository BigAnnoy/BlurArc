# 软件截图存档

本目录用于保存 Blur Arc 的官方软件截图，作为 README、官网落地页、社交媒体、海报配图等场景的素材。

## 文件清单

| 文件名 | 说明 |
|---|---|
| `01-welcome.png` | 首次启动欢迎页：相机图标 + 「欢迎使用 Blur Arc」+ 「选择相册文件夹」主操作按钮 + 隐私本地化提示文案 |
| `02-main-view.png` | 主相册浏览视图：左侧 280px 侧边栏（总文件个数 / 总文件大小 / 照片统计 / 导入照片按钮），右侧照片网格（封面缩略图 + 「选择」入口 + 顶栏主题切换） |
| `03-import-preview.png` | 导入预览对话框：路径 + 总文件个数 / 总文件大小 + 时间线 / 已在相册 / 文件夹内重复 三个标签页 + 日期筛选 + 照片预览 + 「开始导入」操作 |

## 使用建议

- README.md 与 docs/landing.html 中的产品介绍插图建议使用 `02-main-view.png`（最能体现相册管理能力）。
- 强调「本地 / 隐私 / 去重」场景建议使用 `03-import-preview.png`（直接在 UI 内展示了「已在相册 / 文件夹内重复」两个去重视图）。
- 引导安装 / 新手引导场景使用 `01-welcome.png`。
- 海报、宣传物料以本目录截图为基础，结合 `docs/posters/poster.png` 主视觉使用。

## 命名与维护约定

- 编号顺序与产品使用流程一致：欢迎 → 浏览 → 导入。
- 后续新增截图请沿用 `NN-feature-name.png` 命名规范。
- 修改 UI 后请同步更新本 README 中的「说明」一栏。

> 注：若需自动化截图，建议使用 `frontend/` 项目本地 `npm run dev` 启动后，按上述三个场景手动截取，导出为 1x PNG（避免 Retina 模糊）。
