# 侧边栏目录展示方案

## 背景

用户选择的相册路径下，照片可能按不同方式组织：
- 按年月目录（YYYY-MM）存放
- 按自定义文件夹存放（如"旅行照片"、"截图"）
- 直接散放在根目录下
- 多层级嵌套目录结构

侧边栏需要兼容以上所有情况，支持多层级目录浏览，让用户都能方便地浏览照片。

## 目录结构

侧边栏分为两个分组，按条件显示：

```
┌─────────────────────────────────┐
│  📊 统计卡片                     │
│  39 个文件 · 150.8 MB           │
├─────────────────────────────────┤
│  按年份浏览                      │  ← 仅当有 YYYY-MM 目录时显示
│  ▶ 2024                         │
│    1月  50                       │
│    2月  30                       │
│  ▶ 2023                         │
│    12月 20                       │
│                                 │
│  文件夹                          │
│  ▶ 📁 图片 (39)                 │  ← 根目录，第一层
│      📁 屏幕快照 (25)            │  ← 第二层（图片的子目录）
│      📁 Screenshots (14)         │
│      📁 旅行照片 (20)            │
│      📁 截图 (10)                │
│          📁 北京 (12)            │  ← 第三层（截图的子目录）
│          📁 上海 (8)             │
├─────────────────────────────────┤
│  [ 导入照片 ]                    │
└─────────────────────────────────┘
```

## 分组规则

### 1. 按年份浏览

- **显示条件**：相册根目录下存在至少一个 YYYY-MM 格式的子目录
- **分组方式**：按年份归组，年份内按月份排序
- **交互**：点击年份展开/折叠，点击月份加载该月照片
- **排序**：年份降序（最新年份在最上面），月份升序（1月 → 12月）
- **层级**：固定两层（年 → 月）

### 2. 文件夹

- **显示条件**：始终显示（因为根目录始终存在）
- **结构**：多层级可展开的目录树
- **根目录**：作为第一层，显示相册根路径的文件夹名称
- **子目录**：递归展示所有层级的子目录
- **交互**：
  - 点击目录名展开/折叠子目录
  - 点击目录右侧的计数或目录名本身加载该目录下的照片
- **排序**：同级目录按名称排序（中文 locale）

### 3. 空状态处理

- 如果相册根目录下没有任何子目录，根目录项显示为不可展开（无箭头）
- 点击根目录直接加载根目录下的所有照片

## 数据来源

### 后端 `/api/album/tree`

返回相册根目录的完整目录树（纯文件系统扫描），包含：
- `name`：目录名
- `path`：完整路径
- `count`：该目录下（含子目录）的媒体文件总数
- `children`：子目录列表（递归结构）

根节点的 `count` 包含根目录下直接存放的文件数。

### 前端 `api.getTree()`

对后端返回的数据进行分类处理：
1. 遍历根目录的 `children`，将 YYYY-MM 格式的目录归入年月分组
2. 其余目录保留原始树结构，归入普通文件夹列表
3. 根目录的文件数 = 根节点 `count` - 所有子目录的 `count` 之和
4. 返回 `{ tree: YearNode[], rootDir: DirNode | null, regularDirs: DirNode[] }`

其中 `rootDir` 包含完整的树结构（含 children）。

## 根目录入口

- **名称**：使用相册根路径的文件夹名（如 `图片`、`Photos`）
- **路径**：相册根目录的完整路径
- **计数**：根目录下直接存放的媒体文件数（不含子目录中的文件）
- **子目录**：非 YYYY-MM 格式的子目录列表（递归结构）
- **点击行为**：
  - 点击展开/折叠子目录
  - 点击计数或特定区域加载根目录下的照片

## 多层级目录浏览

### 数据结构

```typescript
interface DirNode {
  name: string;
  path: string;
  count: number;  // 该目录下（含子目录）的总文件数
  directCount?: number;  // 该目录下直接存放的文件数（不含子目录）
  children: DirNode[];
}
```

### 展开逻辑

- 使用 `Set<string>` 存储已展开的目录路径
- 点击目录时切换展开状态
- 展开时显示子目录列表，缩进显示

### 加载照片

- 点击目录名加载该目录下的照片（不含子目录中的照片）
- 后端 `/api/album/photos?path=<dir_path>` 只返回该目录直接存放的照片

## 导入后刷新

导入完成后调用 `refreshAppData()`，同时刷新：
- 统计数据（文件数、大小）
- 目录树（年月分组 + 普通文件夹）
- 当前已选目录的照片列表

## 更改相册路径后刷新

更改路径后自动重建索引，完成后调用 `refreshAppData()` 刷新所有内容。

## 实现要点

1. **DirectoryTree 组件**：需要支持递归渲染，每个目录项可展开显示子目录
2. **展开状态管理**：使用 Set 存储已展开的路径，支持多层级同时展开
3. **缩进显示**：每层子目录增加缩进，视觉上体现层级关系
4. **点击区域**：
   - 点击箭头/目录名：展开/折叠
   - 点击计数：加载该目录照片
   - 或者：点击整个目录项加载照片，同时展开（如果未展开）
