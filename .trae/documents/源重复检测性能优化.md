# 源重复检测性能优化计划

## 问题分析

源重复检测在导入预检时执行（`api_server.py` 中的 `_perform_import_check` 方法，第 1151-1172 行），核心逻辑是：

```python
for idx, file in enumerate(source_md5_targets, 1):
    file_path = Path(file['path'])
    md5_hash = calculate_md5(file_path)  # 每个文件都算 MD5
    ...
```

**它在做什么：** 对源文件夹中的**每一个媒体文件**读取整个文件内容并计算 MD5 哈希值（1MB chunk），然后根据 MD5 分组找出重复文件。

**为什么慢：**
1. **全量 MD5**：每个文件都要从头到尾读完算 MD5，即使文件大小差很多也不会跳过
2. **单线程循环**：没有预筛，没有并发，纯串行处理
3. **大文件代价高**：视频文件动辄几百 MB，一个文件就要读很久

## 优化方案：两阶段预筛 + 按需计算 MD5

参考已有的目标重复检测逻辑（第 1174-1288 行），源重复检测也改用预筛机制：

### 第一阶段：轻量预筛（快速）
- 按 `(文件大小, EXIF拍摄时间)` 作为特征键分组
- 只有特征相同的文件才可能重复
- 这一步只需 `stat()` 和读 EXIF，极快

### 第二阶段：按需计算 MD5（精准）
- 只对预筛中特征相同的文件组计算 MD5
- 特征唯一的文件直接跳过 MD5 计算

**预期效果：** 对于大多数不重复的文件，完全不需要计算 MD5，速度提升 5-10 倍。
